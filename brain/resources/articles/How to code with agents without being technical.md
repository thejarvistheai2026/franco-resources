#### Key Details

- kind of build ahead of myself first. I try and just build the thing. And then all of the gaps and all of the issues that I run into are the opportunities for me to learn.

- [agents.md](https://agents.md/) setup for myself because this is effectively like the instruction manual

- what I need to learn is actually **how to work with an AI agent.** How can I prompt it well? How can I make sure it's got the right context? And also how can it help me understand what we're doing, how do the pieces work together, how can I improve my own system over time?

- Another thing while following other engineers is seeing their open source software, cloning it, using it myself, trying to improve it, or just taking parts of it and making that my own. Like Peter's recent summarize YouTube for example, I just took it, removed the Chrome extension part, kept it as the CLI, and now I can just talk to that anywhere I want to.

- So instead I'm coming at it from a point of view of I understand systems thinking for projects built with code.

- ****Every idea you've ever had can be exercised, can be explored, and it doesn't need to be good**.**

- Previously, I'd be too emotionally invested in that idea to be able to just throw it away.

- The way to learn about code is to build ahead of your capability and fail forward.
